Introduction

Hypertext Markup Language (HTML) is a standard markup language that is typically used to create web pages. At least web pages are the main usage for HTML.

HTML provides the (semantic) structure of a "document", the layout is typically done using corresonding Cascading Style Sheets (CSS).

Web browsers read and process HTML to generate web pages.

HTML context

The first version of HTML was specified by Tim Berners-Lee in 1990, one year after he had proposed an Internet-based hypertext system.

The HTML specification became publicly available in 1991.

In the meantime HTML has undergone several major revisions. The following list shows major milestones of the HTML development.

  • November 24, 1995: HTML 2.0 was published.
  • January 14, 1997: HTML 3.2 was published.
  • December 18, 1997: HTML 4.0 was published.
  • October 28, 2014: HTML5 was published.
  • December 14, 2017: HTML 5.2 was published.

Document structuring

HTML uses several different tags for document structuring. The following list shows some examples.

  • <h1>: used for main heading. There should be only one h1 tag per web page. There are further heading tags named h2 to h6.
  • <p>: used for paragraphs in a document.
  • <img>: used to embed images in a document.
  • <div>: used to structure blocks in a document. <div> tags don't get displayed.

Layout considerations

When working with CSS, the respective CSS code can be put at different places:

  • Embedded in an HTML tag: this might make sense in specific cases, yet in most cases maintainability will be reduced by this approach.
  • Embedded in <style> section in an HTML document: preferred over tag-based usage of CSS. Could be used to keep a single file with all document-related content.
  • Included in a separate .css file: in this case the external file gets referenced in the HTML document and document structure and layout are maintained separately. This is the preferred approach that is also leveraged by 3rd-party tools, i.e. as several context-specific .css files can be used.

JavaScript

When an HTML document is loaded into the browser, a so-called Document Object Model (DOM) is being created that contains the document structure and context in a tree-based data structure.

When JavaScript is used in the browser the above mentioned DOM is being used to query and change the HTML document – and thus the web page content.

Following are a couple of JavaScript code samples that show the interaction with the DOM.

document.getElementById("HTML id")

Get the element with the corresponding id attribute.

document.getElementsByClassName("HTML class")

Get all elements with the corresponding class attribute.

element.addEventListener("Event specification")

Adds an event to the specified element.

element.appendChild("Child element")

Append Child element to the specified element.

element.children

Returns a collection of the element's child elements.
